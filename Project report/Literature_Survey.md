# Chapter 5: Literature Survey

---

- In [1], R. Bauder and T. Khoshgoftaar presented a machine learning-based fraud detection system that focuses on analyzing claim patterns to identify fraudulent activities using automated classification based on historical claim data and statistical analysis. However, the system is limited to pattern detection and does not include blockchain immutability or multi-stakeholder verification processes.

- In [2], S. Tanwar, K. Parekh, and R. Evans proposed a blockchain-based healthcare data management system that combines distributed ledger technology with cryptographic security for data integrity and tamper-proof record keeping. The limitation of this approach is high computational resource requirements and lack of integration with machine learning for fraud detection.

- In [3], M. Johnson and A. Williams utilized Natural Language Processing techniques to assess claim descriptions, medical codes, and diagnosis information for generating adaptive risk scores. The effectiveness of this system depends heavily on data quality and relies on predefined fraud pattern models with limited response pattern analysis.

- In [4], J. Chen, P. Kumar, and O. Martinez analyzed various blockchain-based healthcare insurance systems employing smart contracts and distributed ledger techniques for automated claim processing. The limitations include high computational costs, complex infrastructure requirements, and limited practical deployment in healthcare environments.

- In [5], Y. Zhang and F. Liu introduced an AI-based claim verification platform designed for real-time fraud detection scenarios providing instant fraud probability scores. The system lacks holistic security features including blockchain immutability and digital signatures for multi-layer encryption.

- In [6], A. Sharma and R. Patel presented an ensemble-based machine learning approach combining Random Forest, XGBoost, and neural networks achieving high accuracy through feature engineering and model stacking. However, the system lacks explainability features and transparency in decision-making which is not suitable for regulatory compliance requirements.

- In [7], K. Brown and M. Davis focused on detecting collusion patterns between healthcare providers using graph-based network analysis and anomaly detection algorithms for identifying suspicious billing relationships. The limitation is that the system operates in isolation without integration with claim processing workflows and lacks real-time alert mechanisms.

- In [8], S. Nakamoto introduced the foundational blockchain concept through Bitcoin demonstrating peer-to-peer electronic cash system with decentralized consensus mechanism and cryptographic hash chain for immutable transaction records. This work provides the theoretical foundation for implementing blockchain-based audit trails in healthcare systems.

- In [9], L. Breiman proposed the Random Forest algorithm which combines multiple decision trees to achieve robust classification through ensemble learning techniques. This machine learning approach forms the basis for fraud probability scoring in insurance claim analysis with improved accuracy over single classifier models.

- In [10], M.T. Ribeiro, S. Singh, and C. Guestrin presented LIME (Local Interpretable Model-agnostic Explanations) framework for explaining predictions of any classifier enabling transparency in machine learning decisions. This work supports the implementation of Explainable AI features essential for regulatory compliance and stakeholder trust in automated fraud detection systems.
