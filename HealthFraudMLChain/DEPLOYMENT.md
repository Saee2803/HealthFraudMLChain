## Prerequisites

Before deployment, ensure you have:

1. **GitHub Repository**: Your code pushed to GitHub
2. **MongoDB Atlas Account**: For database hosting
3. **Cloud Platform Account**: Render or Railway account

### MongoDB Atlas Setup

1. Go to [MongoDB Atlas](https://www.mongodb.com/atlas)
2. Create a free cluster
3. Create a database user with password
4. Add `0.0.0.0/0` to IP Access List (allows all IPs for cloud deployment)
5. Get your connection string (format: `mongodb+srv://username:password@cluster.mongodb.net/healthfraudmlchain`)

## Deploy on Render

### Method 1: Using render.yaml (Recommended)

1. **Push your code to GitHub**
   ```bash
   cd HealthFraudMLChain
   git add .
   git commit -m "Prepare for deployment"
   git push origin main
   ```

2. **Connect to Render**
   - Go to [Render Dashboard](https://dashboard.render.com/)
   - Click "New +" → "Blueprint"
   - Connect your GitHub repository
   - Render will automatically detect `render.yaml`

3. **Set Environment Variables**
   - After deployment starts, go to your service
   - Navigate to "Environment" tab
   - Add the following variables:
     - `MONGODB_URI`: Your MongoDB Atlas connection string
     - `SECRET_KEY`: (auto-generated by Render)
     - `FLASK_ENV`: `production`

### Method 2: Manual Setup

1. **Create a new Web Service**
   - Go to [Render Dashboard](https://dashboard.render.com/)
   - Click "New +" → "Web Service"
   - Connect your GitHub repository

2. **Configure the Service**
   - **Name**: `healthfraudmlchain`
   - **Root Directory**: `HealthFraudMLChain`
   - **Runtime**: Python
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn main:app --bind 0.0.0.0:$PORT --workers 2 --threads 4 --timeout 120`

3. **Set Environment Variables**
   | Key | Value |
   |-----|-------|
   | `MONGODB_URI` | `mongodb+srv://...` (your connection string) |
   | `SECRET_KEY` | Generate a secure random string |
   | `FLASK_ENV` | `production` |
   | `PYTHON_VERSION` | `3.10.12` |

4. **Deploy**
   - Click "Create Web Service"
   - Wait for the build and deployment to complete
   - Your app URL will be: `https://healthfraudmlchain.onrender.com`

## Deploy on Railway

1. **Go to Railway**
   - Visit [Railway](https://railway.app/)
   - Click "Start a New Project"
   - Select "Deploy from GitHub repo"

2. **Connect Repository**
   - Authorize Railway to access your GitHub
   - Select `HealthFraudMLChain` repository

3. **Configure Service**
   - Railway will auto-detect it's a Python project
   - Set the root directory to `HealthFraudMLChain`

4. **Add Environment Variables**
   - Go to Variables tab
   - Add:
     ```
     MONGODB_URI=mongodb+srv://...
     SECRET_KEY=your-secure-secret-key
     FLASK_ENV=production
     PORT=8080
     ```

5. **Configure Start Command**
   - Go to Settings
   - Set Start Command: `gunicorn main:app --bind 0.0.0.0:$PORT`

6. **Generate Domain**
   - Go to Settings → Domains
   - Click "Generate Domain"
   - Your app will be available at `*.railway.app`

## Start Command

The application uses Gunicorn as the production WSGI server:

```bash
gunicorn main:app --bind 0.0.0.0:$PORT --workers 2 --threads 4 --timeout 120
```

**Options explained:**
- `main:app` - Module and Flask app object
- `--workers 2` - Number of worker processes
- `--threads 4` - Threads per worker
- `--timeout 120` - Request timeout (ML inference may take time)

## Troubleshooting

### Common Issues

#### 1. "ModuleNotFoundError: No module named 'X'"
- Ensure all dependencies are in `requirements.txt`
- Check that the correct Python version is specified

#### 2. "FileNotFoundError: fraud_model.pkl"
- Ensure `fraud_model.pkl` is committed to the repository
- Check that `.gitignore` doesn't exclude `.pkl` files

#### 3. MongoDB Connection Issues
- Verify your MongoDB Atlas connection string
- Ensure `0.0.0.0/0` is in the IP Access List
- Check the database user credentials

#### 4. "Application Error" on startup
- Check the logs in Render/Railway dashboard
- Ensure all environment variables are set
- Verify the start command is correct

#### 5. Static files not loading
- Ensure `static/` folder exists and is not gitignored
- Check file paths use `url_for('static', filename='...')`

### Viewing Logs

**Render:**
- Go to your service → "Logs" tab

**Railway:**
- Go to your service → "Deployments" → Select deployment → "Logs"
